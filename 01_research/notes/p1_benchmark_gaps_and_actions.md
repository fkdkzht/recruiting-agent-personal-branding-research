## 目的

`benchmark_comparison_template.md` に入力済みの **成功3件（P1a）/失敗3件（P1b）**について、Go/No-Go判断に必要な「根拠の密度」を満たすための不足点を洗い出し、次に集める一次情報・確認ポイントをタスク化する。

判断に必要な最低限の観点（要件由来）:
- **市場の受容性**: 「この人に相談したい」を生む約束と信頼根拠が一次情報で説明できるか
- **代表の工数対効果**: 個人依存の度合い、品質再現性、入口の摩擦を推定できるか
- **成約の独自性**: 大手/中堅と違う“指名理由”が何に依存しているか

---

## 次セッションのネクストアクション（推奨順）

この順で進めると、**P3（分析・統合）→P4（検証設計）→P5（レポート）**へ最短で繋がる。

### A. P1の残ギャップを潰し切る（一次根拠の穴埋め）

- **S1（エンジニアのミカタ）**
  - [ ] 数値（平均170万UP／83%／案件数）の **算出条件・母数・期間・例外** の注釈探索（なければ「根拠不明」を明記してP3で地雷化）
  - [ ] 入口（LINE/フォーム）での **期待値調整の文言** 有無（自動返信/注意書き/FAQ等）
  - [ ] 「専任伴走」の一次（担当者像、範囲、例外、返金/解約等）が取れる箇所を回収
- **S2（キッカケエージェント）**
  - [ ] YouTube導線の **一次URL確定**（チャンネル/番組）
  - [ ] LP上の強い言い切り（内定率2倍等）の **注釈/算出根拠の有無** を明確化
  - [ ] 登録/LINE後の **実フロー**（自動返信→面談設定→動画配布 等）を一次で確認
- **S3（Staneer）**
  - [ ] 無料キャリアセッションの **回数・時間・成果物・無料範囲** の一次探索
  - [ ] 所属コンサルの **人数/レビュー・監修体制** の一次探索
- **F3（永信国際）**
  - [ ] 「許認可/欠格/コンプラの根」をHKZ文脈（情報取扱い/利益相反/守秘）に翻訳する素材集め

### B. P3（分析・統合）に着手（いまあるP1を“使える形”に圧縮）

- [ ] `03_analysis/` に **勝ちパターン**（思想→制度、透明な見立て、入口設計、属人性コントロール）を抽出
- [ ] `04_insights/` に **地雷/ガードレール**（信用危機、表示統制、許認可・守秘）を抽出
- [ ] 「HKZに移植するなら何を採用/何を捨てるか」を明文化

### C. P4（検証設計）を具体化（代表ヒアリング前提で確定度UP）

- [ ] `04_insights/concept_test_plan.md` を、P1で得た“刺さる約束/根拠/NG表現”で更新
- [ ] `05_outputs/report_draft.md` の成功基準を、観測指標（具体例が何件でGo寄りか等）に落とす

## P1a（成功）不足点と次アクション

### S1 田代正義 / 株式会社エンジニアのミカタ

- **一次情報の現状**: `sources/2026-04-16_tashiro-engineermikata_sources.md`
- **不足点（判断に必要）**
  - **実績数値の根拠**: 「前職比平均170万円UP」等の数値が、どのページ/条件/期間に基づくか（集計母数・定義が一次情報で取れるか）。
  - **サービスの実態**: 「案件選択」「単価連動」「専任伴走」が、利用フロー上でどこまで制度化されているか（契約形態・条件・例外）。
  - **品質再現性**: 代表個人がどの工程に関与し、組織の仕組みがどこまで担保するか（採用/体制/担当者の役割）。
  - **リスク統制**: 金額訴求/業界批判トーンに対して、炎上・期待差をどう抑える設計か（FAQ/注意書き/表現の幅）。
  - **X（403ブロッカー）**: 代表の発信の主戦場がXなら、手動観測が必要（観測日時と代表投稿の型）。
- **次アクション（具体）**
  - [ ] コーポレートから「実績数値」掲載箇所を特定し、**定義/条件/注意書き**を抜粋（URL＋引用）。
  - [ ] 申込導線（LINE/フォーム）で、**“カジュアル面談”の入口質問/期待値調整**が存在するか確認（可能ならスクショ/文言）。
  - [ ] “専任伴走”のオペに関する一次情報（担当者像、サポート範囲、返金/解約等）を拾う。
  - [ ] Xはブロッカー手順に従い、直近30投稿のうち「思想/制度/CTA」型の代表例を3–5件だけ手動記録。

### S2 IT菩薩モロー（毛呂淳一朗） / キッカケエージェント

- **一次情報の現状**: `sources/2026-04-16_moro-kikkakeagent_sources.md`
- **不足点（判断に必要）**
  - **数値訴求の根拠**: 求人満足度・稼働求人数・年収アップ・内定率などの数値の根拠（定義/比較対象/母数）。
  - **“プロデュース”の実態**: モロー本人がどの程度関与するか（相談・面接対策・監修・登場頻度）。
  - **属人性リスクの扱い**: 強い言い切り（最強/驚異的）に対し、期待値調整や免責があるか。
  - **入口の設計**: 無料登録/LINE登録の直後に何が起きるか（自動返信・面談設定・動画配布の実態）。
- **次アクション（具体）**
  - [ ] トップ/LPの数値訴求について、**注釈（算出条件）**があるかを確認し、なければ「根拠不明」と明記してP3で地雷に接続。
  - [ ] “利用の流れ”から、面談→求人提示→面接対策の粒度（テンプレ/チェックリストの有無）を抜粋。
  - [ ] YouTube導線（番組/チャンネル）を一次情報に追加し、「人格」「専門性」「炎上耐性」観測の入口にする。

### S3 渡邉拓哉 / Staneer（スタニア）

- **一次情報の現状**: `sources/2026-04-16_watanabe-staneer_sources.md`
- **不足点（判断に必要）**
  - **無料キャリアセッションのスコープ**: 何回/何分/何をやる/どこまで無料か（工数対効果の推定に必須）。
  - **“個人ファースト”の具体**: 転職/現職残留の判断をどう支えるか（プロセス・成果物）。
  - **組織化の実態**: 所属コンサルが複数いる前提のため、品質の再現性（採用基準/方法論/レビュー）を一次情報で拾えるか。
  - **抽象価値の検証可能性**: 「熱量を見抜く」等が、具体のケース/評価観点として表現されているか。
- **次アクション（具体）**
  - [ ] 公式サイトで、無料セッションの**条件/回数/所要時間/期待成果物**が書かれている箇所を回収。
  - [ ] Pittaの面談メニューから「入口の型」を抽出（どのテーマが集客しやすいか仮説化）。
  - [ ] “所属コンサル全員が…”の記述について、人数/採用要件/監修体制の一次情報があるか探索。

## P1b（失敗/反面教師）不足点と次アクション

### F1 はんじょう（UUUM所属）/ 信用危機

- **一次情報の現状**: `sources/2026-04-16_failure-hanjou-uuuum_sources.md`
- **不足点（判断に必要）**
  - **本人の一次発信**（説明動画/声明）が公開されているなら、時系列に組み込みたい（“個人の説明”が信頼回復にどう作用するか）。
  - **どの“表現/関与”が引火点になったか**: 反面教師としては「過去関与」だけでなく「拡散の仕組み」を短く抽象化したい。
- **次アクション（具体）**
  - [ ] UUUM声明の「今後の説明（本人YouTube）」が実際に公開済みか確認し、公開済みなら `sources/` に追記。
  - [ ] P3用に、HKZの個人ブランド運用に置き換えたときの「危機管理Runbookの必要要素」を列挙。

### F2 DYM（景表法：優良誤認）

- **一次情報の現状**: `sources/2026-04-16_failure-dym-job-support_sources.md`
- **不足点（判断に必要）**
  - 消費者庁ページに **公表資料（PDF）** がある場合、具体的に「どんな表示がNGだったか」を引用したい（P3/P4のガードレールに直結）。
- **次アクション（具体）**
  - [ ] 消費者庁の公表資料（PDF等）を取得し、NG表示の具体例（文言/数値の示し方）を抜粋。

### F3 永信国際（許可取消）

- **一次情報の現状**: `sources/2026-04-16_failure-eishin-kokusai-license-revocation_sources.md`
- **不足点（判断に必要）**
  - ブランディング論としては「根の前提（許認可/欠格/コンプラ）」を、HKZの文脈（情報取扱い/利益相反/守秘）に翻訳しておく必要がある。
- **次アクション（具体）**
  - [ ] HKZの事業前提として、**職業紹介/派遣に関わるコンプラ論点チェックリスト（最低限）**をP4のガードレールに落とす（外部専門家レビュー前提の“論点一覧”まで）。
