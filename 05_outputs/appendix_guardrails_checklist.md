## 使い方（最小）

- 公開/配布前に、このチェックを **毎回**通す（投稿、LP、資料、動画の概要欄含む）。
- 迷ったら「数値を外す」「断言を弱める」「脚注を足す」「公開を止めてレビュー」に倒す。

---

## A. 表示統制（景表法）公開前チェック（DYMの失敗モード対策）

一次根拠: `01_research/sources/2026-04-16_failure-dym-job-support_sources.md`

### A1. 数値/実績（率・社数・件数・回数・金額）を含む場合

- [ ] **定義**: その数値が「何を数えたものか」を同一コンテンツ内に明記した
- [ ] **期間**: 集計期間を明記した
- [ ] **母数**: 対象人数/件数（n）を明記した
- [ ] **例外**: 除外条件・任意抽出・最大値採用などがある場合に明記した
- [ ] **比較対象**: 「◯倍」「平均との差」等は、比較対象と算出方法を明記した
- [ ] **一次根拠**: 根拠となる一次情報（URL/資料）が社内で辿れる状態にある

### A2. 禁止（原則アウト）

- [ ] 「あたかも」表示になっていない（実態以上に良く見せる断言/誤認誘導がない）
- [ ] 例外があるのに「全称」で断言していない（例: “書類選考なし” など）
- [ ] 社数/求人数などの“現在数”を、定義/カウントルールなしに断言していない

---

## B. 守秘・情報取扱いチェック（候補者/企業情報）

### B1. 外部に出さない（公開物のNG）

- [ ] 実在の企業名/部署名/関係者が特定できる情報を出していない
- [ ] 選考裏情報（通過理由/落ち理由の具体で特定できる情報）を出していない
- [ ] 面談内容/職務経歴などの機微情報を、本人の明示同意なく出していない

### B2. 収集・保管・共有（運用）

- [ ] 収集目的（何のために使うか）を明確化している
- [ ] 共有範囲（誰が見てよいか）を最小化している
- [ ] 保管期間と削除手順（いつ/どう消すか）を決めている
- [ ] 第三者提供の条件（例外）を決めている（原則はしない）

---

## C. 提供範囲・許認可・利益相反チェック（“根”を先に固める）

一次根拠: `01_research/sources/2026-04-16_failure-eishin-kokusai-license-revocation_sources.md`

### C1. 提供範囲の明文化

- [ ] 提供するのが「職業紹介」「募集情報提供」「採用代行（RPO）」「キャリア相談（助言）」のどれかを明記している
- [ ] 境界（やらないこと/できないこと）を明記している
- [ ] 許認可/届出が絡む可能性がある場合、開始前のレビュー（法務）を前提化している

### C2. 利益相反（立場の開示）

- [ ] 報酬構造（どこからお金が発生するか）を社内で把握できている
- [ ] 候補者の利益と衝突しうる場合の開示方針がある
- [ ] 推薦基準/断り方を運用として定義している

---

## D. 事故時初動（Runbook最小形）

### D1. 初動の型（最低限）

- [ ] 初動責任者（誰が判断するか）が決まっている
- [ ] 連絡先（法務/PR/関係者）と連絡順が決まっている
- [ ] 記録（何を残すか：日時/事実/根拠/対応ログ）を決めている
- [ ] 公開/非公開の切り分け基準を決めている

### D2. よくある事故カテゴリ（例）

- [ ] 情報漏えい（候補者/企業の特定につながる）
- [ ] 虚偽/誤認誘導（数値・実績・条件の断言）
- [ ] 許認可・契約・守秘違反の疑い
